Filter

Filters are inserted into filter actions. They are wrapped by { } and then separated by a comma. They can be used to tell a filter what type of items they should look for when filtering.

Syntax- {Properties: Conditional: Equals (Only Needed sometimes)}

Below is the list of properties with the valid conditionals that work for that property.

Properties are separated by : if there are multiple

  1. Name | Title | Author | File Extension | URL | Main Image URL | Number of Words | Excerpt | First Name | Middle Name | Last Name | Phone Number | Email Address | Street Address | Prefix | Suffix | Nickname | Phonetic Last Name | Phonetic Middle Name | Company | Job Title | Department | Notes | Location
    • Conditionals: ==, != , .contains, !contains, .beginsWith, .endsWith
    • Syntax - {Name: == : Zachary}
  2. File Extension
    • Conditionals: ==, != , .contains, !contains, .beginsWith, .endsWith
    • Syntax - {File Extension: .contains : .jelly}
  3. File Size
    • Conditionals: ==, != , .contains, !contains, >, >=, <=, <
    • Acceptable extensions for sizes Bytes, KB, MB, GB, TB, PB, EB, ZB. Size is placed in equal.
    • Syntax - {File Size: == : 1 gb}
  4. Last Modified Date | Published Date | Creation Date | Date Taken | Time Take | Due Date
    • Conditionals: ==, != , .after, .before, .today, .between, .last
    • Acceptable extensions for the property .last: years, months, weeks, days, hours, minutes, seconds
    • Syntax - {Creation Date: .last : 2 years, Last Modified Date: .today}
  5. Status
    • Conditionals: Is, Is Not
    • Excepted Statuses: Accepted, Completed, Declined, Delegated, In Process, Pending, Tentative, Unknown
    • Syntax - {Status: Is Me, Status: Is, Accepted}
  6. Role
    • Conditionals: Is me, Is Not Me, Is, Is Not
    • Excepted Roles: Unknown, Required, Optional, Chair, Non-Participant
    • Syntax - {Role: Is : Chair, Role: Is Not Me, Role: Is Not: Unknown}
  7. Width | Height | Duration | Frame Rate
    • Conditionals: ==, !=, >, >=, <=, <
    • Syntax - {Q: > : 30}
  8. Orientation
    • Up, Down, Left, Right, Up Mirrored, Down Mirrored, Left Mirrored, Right Mirrored
  9. Is | Is Not | Is Hidden | Is Not Hidden | Is Favorite | Is Not Favorite | Is Me | Is Not Me | Is Screenshot | Is Not a Screenshot | Has Photo | Has No Photo | Is All Day | Is Not All Day | Has Alarms | Has No Alarms | Has Subtasks | Has No Subtasks | Is Flagged | Is Not Flagged | Is Completed | Is Not Completed
    • Used as a boolean.
    • Syntax:{Is Hidden}
  10. Media Type
    • Image, Video, Audio
    • Syntax - {Media Type: Is: Image}
    • You can also include any variable
  11. Photo Type
    • HDR, Panorama, Burst, Live Photo
    • Syntax - {Photo Type: Is Not: Live Photo}
    • You can also include any variable
  12. Group
    • This excepts a contact group. To denote a contact group use c-•group name•. If you want to use a variable instead just put the variables name.
    • Syntax - {Group: Is: c-Family}
  13. Folder
    • This excepts a folder group. To denote a folder name use f-•folder name•. If you want to use a variable instead just put the variables name.
    • Syntax - {Folder: Is: f-Friends}
  14. Album
    • This excepts a photo album. To denote a photo album use a-•album name•. If you want to use a variable instead just put the variables name.
    • Syntax - {Album: Is Not : a-Wallpapers}
  15. Calendar
    • This excepts a calendar. To denote a calendar use c-•calendar name•. If you want to use a variable instead just put the variables name.
    • Syntax - {Calendar: Is : c-School}
  16. Priority
    • Low, Medium, High
    • Syntax - {Priority: Is : High}
    • You can also include any variable.
