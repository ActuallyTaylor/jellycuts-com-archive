Jelly Version 2.0 update notes

The Jelly Programming Language

The Jelly programming language was created to code Siri Shortcuts in a text base language. This is why the language is currently structured around calling functions.

The Shortcuts Library

The standard Jelly library is the Shortcuts Library. This library contains functions that are easily translated directly into default Shortcuts actions. Documentation for this library can be found on the Shortcuts Standard Page.

Hello all! Welcome to the documentation for Jelly version 3.0, this page will go over all of the updates for changes that come with a major language update. I recommend reading as much of this page as possible as it will make your transition to the new language version much easier and significantly quicker.

Syntax Changes

Variables

The biggest change since version 2.0 is how variables are named. In previous versions variables could be named with spaces. In Jelly 3.0 this has been changed to more closely match other programming languages that do not allow spaces in variable names. The characters that are allowed in variables are as follows [a-z][A-Z][0-9]_$.

Setting Variables

You can set variables by using the following syntaxes:


var x = 3
                
var y = "Hello World!"
                
var z = ShortcutInput

Updating A Variable

You can update the value of a variable using the following syntaxes:


x = 3
                
y = "Hello World!"
                
z = ShortcutInput

Functions

Previously in Jelly 2.0 you were able to call functions without naming any parameters. However, in Jelly 3.0 parameters are now required for functions. 

text(text: "Hello World!")

Menu Control Flows

Menus have been altered to match switch statements from other languages. In previous versions of Jelly you would need to list all case names in a menu's header. You now no longer need to do this. The only thing needed in the head statement is the menu prompt.

Another change to menu statements is that case statements no longer need parentheses. They now only need the name of the option surrounded by parentheses.


menu("Hello World") {
    case "Case One":
        text(text: "Test")
    case "Case Two":
        text(text: "Test Two")
}

Conditional Control Flows

If statements have also been altered to be easier to type on a mobile device along with matching the Swift language closer. This means that an if statement can look like either of these:


if ShortcutInput.as(Text) == "Hello World" {
    quicklook(input: ShortcutInput)
}
                
or
                
if (ShortcutInput.as(Text) == "Hello World") {
    quicklook(input: ShortcutInput)
}
